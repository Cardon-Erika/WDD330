Chapter 11 - Further Functions

Immediately Invoked Function Expressions (IIFE)

An IIFE is an anonymous function that is invoked as soon as it's defind. They are a useful way of performing a task while keeping any variables wrapped up within the scope of the function. This means the global namespace is not polluted with lots of variable names.

IIFE's can be used to set up initialization code. Because the code is run only once, there's not need to create any reusable, named functions, and all the variables will also be temporary.

Much of this can be done in ES6 now however, by simply placing the code inside a block.

Recursive Functions

A recursive function is one that invokes itslef until a certain condition is met. It's a useful tool to use when iterative processes are involved. A commmon example is a function that calculates the factoral of a number.

Promises

A promise represents the future result of an asynchronous operation. Promises don't do anything that can't already be achieved using callbacks, but they help simplify the process, and avoid the convoluted code that can result from using multiple callbacks.

When a promise is created, it is said to be pending. It reminas in this state while the operation is taking place. At this stage it is unsettled. Once the operation has completed, the promise is said to have been settled. A settled promise has two possible outcomes: resolved or rejected.

Chaining Multiple Promises

Promises come into their own when multiple asynchronous tasks are required to be carried out one after the other. If each function that performs an async operation returns a promise, we can chair the then() methods together to form a sequential piece of code that's easy to read. Each promise will only begin once the previous promise has been settled.

Pure Functions

A key aspect of functional programming is its use of pure functions. A pure function is a function that adheres to the following rules:

  1. The return value of a pure function should only depend on the values provided as arguments. It doesn't rely on values from somewhere else in the program.
  2. There are no side-effects. A pure function doesn't change any values or data elsewehere in the program. It only makes non-destructive data transformations and returns new values, rather than altering any of the underlying data.
  3. Referential transparency. Given the same arguments, a pure function will always return the same result.

Chapter 13 - AJAX

AJAX

AJAX is a technique that allows web pages to communicate asynchronously with a server, and it dynamically updates web pages without reloading. This enables data to be sent and received in the background, as well as portions of a page to be updated in response to user events, while the rest of the program continues to run.

Basic Usage

The Fetch API provides a global fetch() method that only has one mandatory argument, which is the URL of the resource you wish to fetch.

Reponse Interface

The Fetch API introduced the Response interfave that deals with the object that's returned when the promise is fullfilled. This means that we can use an if block to check if the request was successful, and throw an error otherwise.

JSON Response

JSON is probably the most common format for AJAX responses. The json() method is used to deal with these by transforming a stream of JSON data into a promise that resolves to a JavaScript object.

Chapter 12 - Canvas, SVG, & Drag and Drop

Canvas vs. SVG

Canvas allows for pixel manipulation. However, cnavas operates in "immediate mode", meaning if you can't add to what is already in the canvas once you finish drawing a shape. You must redraw it from scratch. Canvas also isn't accessable from the DOM. That makes canvas much faster than SVG though. Canvas is generally the better choice if you're looking to design a game requireing lots of animations.

By contrast, SVG is accessable from the DOM, because the structure of the image is preserved in the XML document that describes it. SVG also has a more complete set of tools to help you work with it. However, since SVG is a file format, rahter than a set of methods that allows you to dynamically draw on a surface, you can't manipulate SVG images the same way you can manipulate pixels on canvas. For example, with canvas you can make a colored image appear black and white, but that would not be possible with SVG.

In summary, if you need to paint pixels to the screen and have no concerns about the ability to retrieve and modify your shapes, canvas is probably the better choice. If, on the other hand, you need to be able to access and change specific aspects of your graphics, SVG might be more appropriate.
